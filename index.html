The actual mandate

Reduce the distance between AI ambition and a production-capable workflow.

Deloitte’s FDE model asks one practitioner to move fluidly between discovery, software, architecture, evaluation, stakeholder alignment, production hardening and capability transfer.

Move at frontier-model speed while delivering at enterprise-control standards.
01

Speed vs. governance

Prototype quickly without postponing permissions, failure behavior, economics or ownership.

02

Client specificity vs. reusable IP

Solve the actual workflow while retaining patterns that improve the next engagement.

03

Model capability vs. system reliability

A powerful response is not a production system. Repeated behavior and recovery matter.

04

Autonomy vs. human authority

Autonomy must be earned by consequence, evidence, permissions and reversibility.

Forward Deployment Contract

Advance the workflow only when its next operating obligation is satisfied.

01

Frame

Changed work, affected user, baseline, constraints and accountable owner.

Exit: a workflow problem—not an AI feature request.
02

Prove

The smallest end-to-end implementation testing fit, behavior and usefulness.

Exit: a real user completes meaningful work.
03

Bound

Permissions, approvals, refusal, evaluation, cost, latency and model risk.

Exit: allowed, prohibited and held actions are explicit.
04

Embed

Identity, APIs, logging, observability, CI/CD, support and cadence.

Exit: capability works inside the real workflow.
05

Transfer

Runbook, training, ownership, architecture record, backlog and reusable assets.

Exit: the client can operate and improve it.

First 90 days

Earn technical trust, own a bounded workstream, then prove transferability.

Days 1–30

Learn the pod and ship

  • Learn approved platform, architecture, security, evaluation and code standards.
  • Pair on client framing and working sessions.
  • Build small tool-use and retrieval implementations.
  • Target: first merged engineering contribution.
Days 31–60

Own a bounded workstream

  • Define workflow baseline and value hypothesis.
  • Implement one end-to-end AI-enabled workflow.
  • Define permissions, approvals and failure tests.
  • Target: representative users complete meaningful work.
Days 61–90

Prove operability and transfer

  • Support production readiness and CI/CD decisions.
  • Create runbook, monitoring, escalation and ownership.
  • Pair with client engineers and operators.
  • Target: client-operable workflow and reusable Deloitte capability.
